## Headline

**RC2's headline deliverable shipped**: `crabscheme aot prog.scm
--build` produces a standalone native binary from a Scheme source
file.

```bash
$ echo '(define (fact n) (if (= n 0) 1 (* n (fact (- n 1)))))' > fact.scm
$ crabscheme aot fact.scm --build
crabscheme aot: emitted project at fact-aot
  built: fact-aot/target/release/fact

$ ./fact-aot/target/release/fact 10  # → 3628800
```

The pipeline composes: cs-parse → cs-expand → cs-vm::compile →
cs-vm::jit_translate::bytecode_to_rir_aot → cs-aot::project::
emit_project → cargo build → native binary.

## What's blocking 1.0-rc2 tag

Nothing technical — the user-visible improvements (let support,
CLI flags, doc + bench scorecard) are merge-ready.

Open question: should we ship 1.0-rc2 with the current AOT
coverage (2/8 microbenches, single-define Scheme programs), or
wait for iter K (MakeClosure) which would push it to 6+/8?

Argument for shipping now: RC2 already strictly improves on RC1
across many axes; users with self-recursive numeric kernels get
real value. Argument for waiting: "crabscheme aot" is most
compelling when it handles arbitrary Scheme; today it handles a
documented subset, which feels half-done in the README.

## Path to 1.0-rc2 (recommended order, if shipping after iter K)

1. **iter K** (heavy): MakeClosure + general Call. Needs:
   - cs-vm to expose a public Procedure-heap API (currently the
     proc_table is internal).
   - cs-aot to emit allocation + dispatch.
   - End-to-end test with a closure-using Scheme program.
2. **iter Q**: Top-level program synthesis. Builds on K.
3. **iter U**: EnvLookup/Set lowering for cases demote can't fix.
   Needs cs-vm JIT_CALLER_ENV install API.
4. **Tag `1.0-rc2`** + run the release workflow.

## Known limitations (carry-forward from RC1)

- AOT-NB binaries require cs-vm at a path-resolved location. A
  release-installed binary (from the 1.0-rc1 GitHub release)
  can't AOT for users — cs-vm has to be reachable. iter Y
  (crates.io publish) fixes this; it's deferred because publishing
  is its own release-engineering arc.
- Browser WASM (`wasm32-unknown-unknown`) not supported.
- Multi-shot `call/cc` only on VM tier (M8); walker-tier capture
  remains deferred.
- Canonical Larceny / Racket-R6RS suite imports — gate is "MET on
  our 117-fixture corpus + sample" per the 1.0-rc-readiness doc;
  full canonical-suite imports remain post-1.0 measurement-infra
  work.
